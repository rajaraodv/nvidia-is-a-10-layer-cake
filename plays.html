§ 1 · Play 01
PLAY 01 · QUARTER
Ship the Nemotron Cookbook
A standalone, framework-agnostic, copy-pasteable recipe site that an indie dev with a MacBook can follow.
Why it works
claude-cookbooks and openai-cookbook are the most-cited developer assets either company has. Notebook-first, scenario-organized, runnable in one click on Colab or locally. They became the unofficial "how to use this model" canon.
The gap today
NVIDIA's recipes live inside NVIDIA-NeMo/Nemotron and assume the full NVIDIA stack — NIM, CUDA, TensorRT-LLM. An indie dev with a Mac can't follow them. 80% of the surface is unreachable.
Competitor proof
Anthropic's claude-cookbooks is cited as best-in-class. OpenAI's openai-cookbook is the format every other lab copies.
What "winning" looks like
github.com/NVIDIA/nemotron-cookbook with 5K+ stars by GTC DC 2026. Cited in Latent Space / AI Engineer. Indie devs link to it on Twitter when they ship something with Nemotron.
What we'd build
  • New standalone GitHub org: github.com/NVIDIA/nemotron-cookbook
  • 20–30 runnable notebooks: agent loops, RAG, structured outputs, fine-tuning, multimodal, voice — 80% framework-agnostic, 20% NVIDIA-stack-optimized recipes
  • One-click Colab and Brev launchers per notebook
  • Same content syndicated to build.nvidia.com/cookbook as the canonical home
  • Quarterly "Cookbook spotlight" — featured community recipes from external devs
OwnerThis role · DevRel · Product
PartnersCursor, LangChain, OpenRouter for recipe contributions
Time to ship~10 weeks for v1 (20 recipes)
Lead metricGitHub stars · weekly clones · external blog citations
§ 2 · Play 02
PLAY 02 · QUARTER
Buy default IDE placement — start with Cursor
Cursor is already a Coalition member contributing benchmarks. Convert that into "Nemotron is the default model new Cursor users see."
Why it works
IDE / agent-tool defaults are the single highest-leverage developer distribution channel of 2026. Owning even one Cursor / Windsurf / Zed / Claude Code default reaches tens of millions of monthly developer impressions.
The gap today
Cursor is in the Nemotron Coalition and contributes coding benchmarks. Yet Nemotron is not in Cursor's default model dropdown. Anthropic is. The Coalition asset is unconverted.
Competitor proof
Claude is the default in Cursor + powers Windsurf's Cascade + integrates with Zed. It's how Anthropic built developer mindshare without owning a consumer product.
What "winning" looks like
Nemotron is Cursor's free-tier default for new users in three target use cases (e.g., agent loops, structured outputs, fast coding). Wired up before GTC DC. Then Windsurf, then Claude Code, then Zed.
What we'd build
  • Co-marketing deal with Cursor — explicit free-tier default routing for new users on Nemotron 3 Super
  • Joint launch: "Cursor + Nemotron — fast, open, frontier coding free for every new dev"
  • Cursor-tuned recipes in the Cookbook (Play 01)
  • Telemetry deal: track activation, retention, and code completion metrics for the cohort
  • Roadmap: Windsurf (Q+1), Claude Code partnership exploration, Zed (open-source play)
OwnerThis role · BD · Strategic Partnerships
PartnersCursor · then Windsurf · then Zed
Time to ship~12 weeks for the Cursor deal
Lead metricDaily active Nemotron requests from IDE-source · IDE-cohort retention
§ 3 · Play 03
PLAY 03 · QUARTER
Activate the founder / researcher X voice
Stop launching Nemotron from corporate handles. Land it from Bryan, Mostofa, Kalpit and the people who actually built it.
Why it works
Developer Twitter rewards named, technical, founder-voice. Dario, Mike Krieger, Mensch, DeepSeek researchers all drop benchmarks themselves. Corporate handles get retweets; researcher accounts get virality, dev-press follow-on, and trust.
The gap today
@NVIDIA and @NVIDIAAIDev are corporate-coded — slow cadence, no benchmark drops, no "weights are live" energy. The Nemotron researchers exist on X but aren't on the launch front line.
Competitor proof
DeepSeek's whole 2025-26 trajectory was researcher tweets going viral. Anthropic launched Claude Opus 4.6 with a Dario thread. Mistral 3 was a Mensch + ASML thread.
What "winning" looks like
A roster of 5–8 named NVIDIA AI researchers with active developer-Twitter presence. Every Nemotron release gets at least 3 founder/researcher threads with benchmark screenshots within hours. Trending in developer circles for the model — not for the GPU.
What we'd build
  • Recruit 5-8 named NVIDIA AI leads as "Nemotron faces" — incentivized to post benchmark drops, training tricks, "weights live" threads
  • Internal X studio: prep benchmark screenshots, recipe shorts, response templates so researchers can post quickly without losing voice
  • Weekly "release rhythm" — one researcher post per business day from Nemotron leads, escalating during launches
  • Founder-thread template playbook for every Nemotron release (3 threads, 3 angles, 3 voices)
  • Tie-in to the Cookbook (Play 01) — researcher posts link to runnable notebooks

§ 4 · Play 04
PLAY 04 · HALF
Go MCP-native. Ship 20+ first-party MCP servers.
If MCP is the lingua franca of the agent era, NVIDIA's absence from it is unforced.
Why it works
MCP is the standard for tool-calling and agent integrations in 2026. Cursor, Windsurf, Zed, VS Code, Claude Code, ChatGPT Apps SDK — all MCP-compatible. Every model not MCP-native feels legacy.
The gap today
NVIDIA has no first-party MCP servers for the Nemotron / NIM / Triton / Run:ai / AI Workbench / Brev stack. An agent developer reaching for "the MCP server for $X NVIDIA product" finds nothing.
Competitor proof
Anthropic invented MCP. OpenAI extended it via ChatGPT Apps SDK. Thomson Reuters CoCounsel ships an MCP server. 9 creative-tool connectors (Blender, Adobe, Autodesk, Ableton, Splice) ship MCP integration with Claude.
What "winning" looks like
20+ first-party MCP servers on the official MCP registry by end of Q3 — Nemotron, NIM deploy, NGC pull, Triton serve, Run:ai schedule, Workbench env, Brev launchable, DGX Cloud, build.nvidia.com playground, plus model-specific (Parakeet voice, GR00T sim, Cosmos generate, etc.).
What we'd build
  • MCP server reference architecture for the NVIDIA stack — versioned, on the official registry
  • 20+ first-party servers across products (priority: model invocation, NIM lifecycle, NGC pull, Triton serve, Run:ai schedule, Brev launchable)
  • Sample agent built on those MCP servers — included in the Cookbook (Play 01)
  • Sponsored MCP server contributions for top OSS tools (a Cursor MCP server using Nemotron · a LangChain MCP integration · a llama.cpp port)
  • Joint launch with Anthropic: "Claude + Nemotron Coalition MCP servers ship together"
OwnerThis role · Engineering · DevRel
PartnersAnthropic (MCP registry) · Cursor · LangChain
Time to ship~6 months for 20 servers + sample agent
Lead metricMCP servers shipped · installs on MCP registry · agent integrations citing NVIDIA servers
§ 5 · Play 05
PLAY 05 · HALF
Launch the $50M "Built with Nemotron" program
Inception covers startups. Nothing covers the indie devs and OSS maintainers writing the missing "How I built X with Nemotron" stories.
Why it works
"How I built X" content on Hacker News, r/LocalLLaMA, dev.to, Substack — that's the surface that proves a model is real. It's also the cheapest paid-distribution channel in dev marketing.
The gap today
No equivalent of Anthropic's $100M Partner Network for indie devs and OSS maintainers building on Nemotron. The LangChain integration writer, the llama.cpp porter, the Cursor extension dev, the r/LocalLLaMA hobbyist — nobody is paid to ship Nemotron content.
Competitor proof
Anthropic's $100M Claude Partner Network (March 2026) — direct dollars, training, sales enablement. Accenture trained 30K. Deloitte 470K seats. The dollars create the partners; the partners create the proof.
What "winning" looks like
500 "Built with Nemotron" projects shipped within 12 months. 50 with HN front-page write-ups. 5 case studies of the year from indie devs. Nemotron mentioned in the same breath as Llama and Qwen on r/LocalLLaMA.
What we'd build
  • $50M committed across 24 months — $30M direct grants to indie devs, $10M to OSS maintainers (LangChain integration, llama.cpp port, vLLM kernels, Cursor extensions), $10M Latent Space / AI Engineer / KubeCon sponsorships + speaker support
  • Grant tiers: $5K micro-grants (rapid, 50+ per quarter), $25K project grants, $100K studio grants for ambitious builds
  • Co-promotion: every grantee gets a Cookbook recipe (Play 01) + a researcher boost on Twitter (Play 03)
  • "Built with Nemotron" landing page — gallery, sortable by use case
  • Annual "Builders Day" — physical event in SF + virtual, all NVIDIA grantees pitch their projects
